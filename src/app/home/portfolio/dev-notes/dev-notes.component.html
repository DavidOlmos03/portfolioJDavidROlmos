project_name->database->migration

Nos permite crear el esquema y las tablas de la base de datos, se reconoce por el Schema::create()... De esta forma se crean automaticamente en la base de datos sin necesidad de crearlas a mano.
Obs. Nombre de tablas en plural y nombres de modelos en singular

Nuevo modelo

project_name->app->model Nos permiten interactuar con la base de datos, esto nos permite escribir $query
$fillable contiene los campos que se podan ingresar a la base de datos

Documentación relaciones

No es necesario llamar el namespace del modelo a utilizar en otro modelo, pues estando en el mismo directorio esto no es necesario.

Para una tabla pivote no es necesario crear un modelo, solo se crea la migración, cuando creo las relaciones inversas en dos modelos Laravel entiende que se utiliza una tabla pivote y la busca.

Documentación Laravel breeze

Se utiliza este paquete para crear un sistema de autenticación completo, se selecciona el motor de plantillas blade de Laravel, su frontend esta creado con Tailwind CSS.

En routes/web.php estan las rutas con sus respectivos nombres, las cuales apuntan a una vista o a un controlador.
En la ruta llamo al controlador y dentro de este creo una función que me retorna la vista, estas vistas se pueden crear con blade.
